Интеграция ЭЦП (Электронная Цифровая Подпись)

1 Клиент (Браузер)

JavaScript клиент подключается к NCALayer через WebSocket (ws://127.0.0.1:13579)

2 NCALayer (Локально)

Приложение NCALayer на компьютере пользователя обрабатывает запросы на подпись

3 Django Backend

Генерирует challenge, проверяет подпись через NCANode API, авторизует пользователя

4 NCANode API

Внешний сервис для проверки подписи и валидации сертификата

Успех

Ключевые компоненты:

  • ecp_challenge - генерация уникального challenge (nonce)
  • ecp_verify - проверка подписи и авторизация
  • Извлечение ИИН/БИН - из SubjectDN сертификата
  • Автоматическое создание пользователя - при первом входе

Модель данных для ЭЦП

UserProfile (Расширена)

  • iin - ИИН (для физических лиц)
  • bin - БИН (для юридических лиц)
  • certificate_serial - Серийный номер сертификата
  • ecp_enabled - Флаг использования ЭЦП

Безопасность

  • Challenge с ограниченным сроком (5 минут)
  • Криптографически стойкая генерация
  • Проверка OCSP и CRL
  • Логирование всех операций

Процесс авторизации:

1. Пользователь выбирает сертификат ЭЦП → 2. Подписывает challenge → 3. Сервер проверяет подпись → 4. Извлекает ИИН/БИН → 5. Находит или создает пользователя → 6. Авторизует в системе

Безопасность и валидация

ЭЦП Безопасность

  • Криптографически стойкий challenge (secrets.token_urlsafe)
  • Ограниченный срок действия (5 минут)
  • Проверка OCSP и CRL
  • Валидация через NCANode API
  • Логирование всех операций

Общая безопасность

  • CSRF защита
  • Session-based аутентификация
  • HTTPS в production
  • Валидация входных данных
  • Rate limiting (планируется)

Обработка ошибок:

  • NCALayer не установлен / не запущен
  • Сертификат не найден или истек
  • Недействительная подпись
  • Истекший challenge
  • Ошибки NCANode API
